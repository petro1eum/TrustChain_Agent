# Create File Tool

Инструмент для создания новых файлов в Docker контейнере.

## Параметры

- `description` (required, string) - Зачем создаю этот файл (ОБЯЗАТЕЛЬНО, указывай ПЕРВЫМ)
- `path` (required, string) - Абсолютный путь к создаваемому файлу в контейнере (ОБЯЗАТЕЛЬНО, указывай ВТОРЫМ)
- `file_text` (required, string) - Полное содержимое файла (ОБЯЗАТЕЛЬНО, указывай ПОСЛЕДНИМ)

## КРИТИЧЕСКИ ВАЖНО: Порядок параметров

Параметры должны быть указаны в строгом порядке:
1. `description` - ПЕРВЫМ
2. `path` - ВТОРЫМ
3. `file_text` - ПОСЛЕДНИМ

## Директории для файлов

- `/home/kb` - временная рабочая директория (сбрасывается между задачами)
- `/mnt/user-data/outputs` - финальные файлы для пользователя (ОБЯЗАТЕЛЬНО копируй сюда готовые файлы!)

## Примеры

```json
{
  "description": "Создаю Python скрипт для обработки CSV",
  "path": "/home/kb/process_data.py",
  "file_text": "import pandas as pd\n\ndf = pd.read_csv('data.csv')\nprint(df.head())"
}
```

```json
{
  "description": "Создаю конфигурационный файл",
  "path": "/home/kb/config.json",
  "file_text": "{\n  \"debug\": true,\n  \"timeout\": 30\n}"
}
```

## Работа с длинными файлами

Для файлов >100 строк используй итеративное редактирование:
1. Создай структуру через `create_file`
2. Добавляй контент по частям через `str_replace`
3. Копируй финальную версию в `/mnt/user-data/outputs`

## Результат

Возвращает:
- `success` - статус выполнения
- `path` - путь к созданному файлу
- `size` - размер файла в байтах
- `message` - сообщение о результате
