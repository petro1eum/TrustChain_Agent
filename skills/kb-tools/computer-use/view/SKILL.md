# View Tool

Инструмент для просмотра содержимого файлов, изображений и директорий в Docker контейнере.

## Параметры

- `path` (required, string) - Абсолютный путь к файлу или директории в контейнере
- `view_range` (optional, array) - Диапазон строк для текстовых файлов [start, end]. Используй [-1] для просмотра до конца файла
- `description` (required, string) - Зачем просматриваю этот файл/директорию (ОБЯЗАТЕЛЬНО)

## Поддерживаемые типы

- **Директории**: список до 2 уровней вглубь
- **Изображения**: .jpg, .jpeg, .png, .gif, .webp
- **Текст**: любые текстовые файлы с нумерацией строк

## Важные правила

1. **ВСЕГДА указывай `description`** - объясни зачем просматриваешь файл
2. **Используй `view_range`** для больших файлов - не загружай весь файл сразу
3. **Путь должен быть абсолютным** - начинай с `/`

## Примеры

```json
{
  "path": "/mnt/skills/public/docx",
  "description": "Проверяю структуру docx skill"
}
```

```json
{
  "path": "/mnt/skills/public/docx/SKILL.md",
  "view_range": [1, 50],
  "description": "Читаю первые 50 строк docx skill"
}
```

```json
{
  "path": "/home/kb/data.csv",
  "view_range": [1, 100],
  "description": "Проверяю формат данных перед обработкой"
}
```

## Результат

Возвращает:
- `success` - статус выполнения
- `type` - тип просмотренного объекта (file, directory, image)
- `path` - путь к объекту
- `content` - содержимое (для текстовых файлов)
- `lines` - количество строк (для текстовых файлов)
- `view_range` - использованный диапазон
